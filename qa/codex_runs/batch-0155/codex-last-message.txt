            <h3>Binary as a positional system (base 2)</h3>
            <p>Like decimal, the <strong>binary system</strong> is positional: each digit (bit) has a weight that depends on its position. In base 10 the weights are powers of 10 (1, 10, 100, …); in base 2 they are powers of 2:</p>
            <div class="formula-box">
              <p>For a binary number ⁠b<sub>n-1</sub> … b<sub>1</sub> b<sub>0</sub> with bits b<sub>i</sub> ∈ {0,1}:</p>
              <p>(b<sub>n-1</sub> … b<sub>1</sub> b<sub>0</sub>)<sub>2</sub> = ∑<sub>i=0</sub><sup>n-1</sup> b<sub>i</sub> 2<sup>i</sup>.</p>
            </div>
            <p>Example: the binary number 101101<sub>2</sub> equals</p>
            <div class="formula-box">
              <p>1·2⁵ + 0·2⁴ + 1·2³ + 1·2² + 0·2¹ + 1·2⁰ = 32 + 0 + 8 + 4 + 0 + 1 = 45<sub>10</sub>.</p>
            </div>
            <h3>Hexadecimal and octal as compact binary</h3>
            <p>Binary is natural for machines but verbose for humans. Two derived bases are used as compact notations:</p>
            <ul>
              <li><strong>Hexadecimal (base 16)</strong> uses digits 0–9 and letters A–F for values 10–15. One hex digit corresponds to <strong>4 bits</strong>.</li>
              <li><strong>Octal (base 8)</strong> uses digits 0–7. One octal digit corresponds to <strong>3 bits</strong>.</li>
            </ul>
            <p>Grouping binary digits (bits) into 4-bit nibbles or 8-bit bytes builds the mental bridge between binary and hex used in debuggers and low-level tooling.</p>
            <h3>Binary arithmetic in a nutshell</h3>
            <p>Binary arithmetic obeys the same rules as decimal, but carries occur at powers of 2:</p>
            <div class="formula-box">
              <p>Addition table:</p>
              <ul>
                <li>0 + 0 = 0</li>
                <li>0 + 1 = 1</li>
                <li>1 + 0 = 1</li>
                <li>1 + 1 = 10<sub>2</sub> (write 0 and carry 1)</li>
              </ul>
            </div>
            <p>Higher-level operations (subtraction, multiplication, division) reduce to the same algorithms you know from decimal, but carried out bit by bit. In practice, programmers often rely on built-in integer types and bitwise operators, while using tools like this calculator to inspect and reason about <strong>binary representations</strong>.</p>
            <h3>Text and binary: bytes, codes and encodings</h3>
            <p>Computers do not “store letters”, they store bytes. A byte is typically <strong>8 bits</strong>, which can represent integers from 0 to 255. Text encodings (ASCII, UTF-8 and others) map these integers to characters.</p>
            <div class="formula-box">
              <p>For example, in ASCII/UTF-8:</p>
              <ul>
                <li>'A' → 65 → 01000001<sub>2</sub></li>
                <li>'a' → 97 → 01100001<sub>2</sub></li>
                <li>'0' → 48 → 00110000<sub>2</sub></li>
              </ul>
            </div>
