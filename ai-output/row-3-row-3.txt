1) Thinking Process (UX & IA)

- UX archetype: This is a simple, single-purpose unit converter with minimal inputs, so I’m using a **Wizard-lite / simple form** rather than a dense cockpit. Inputs and results are shown **side-by-side on desktop** and **stacked on mobile** via layout classes handled inside `CalculatorIsland`.
- Progressive disclosure: Only a **Basic** mode is needed (kilograms to pounds). An optional **Advanced** toggle is provided for reverse conversion (pounds to kilograms) and extra precision controls, but kept collapsed by default.
- Inputs: 
  - Primary input: **Weight in kilograms** with `inputmode="decimal"` and clear label.
  - Optional: **Number of decimal places** for result precision in Advanced mode.
- Results:
  - Main result: **Pounds** with large, high-contrast typography.
  - Sticky result bar on mobile is handled inside `CalculatorIsland` (per project standard).
- Microinteractions:
  - Result highlight on change (handled in island).
  - Tooltips for “What is a kilogram?” and “What is a pound?” plus precision explanation.
- SEO & IA:
  - URL: `/en/science-and-mathematics/unit-conversion/kg-to-lbs`
  - Rich content: theory section with formulas in LaTeX-style text, real-world example, and 3 targeted FAQs.
  - JSON-LD: `WebApplication`, `FAQPage`, and `BreadcrumbList`.
  - Application category: `EducationalApplication` (science/math unit conversion).
- Related tools:
  - Same cluster: other unit conversion tools (e.g., meters to feet, Celsius to Fahrenheit) referenced generically in `relatedTools` for cross-linking, but not invented beyond simple slugs consistent with the cluster.
