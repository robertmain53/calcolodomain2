1) Thinking Process

- UX archetype: This is a simple, fast unit conversion with minimal inputs, so I’m using a **Wizard-lite / simple form** rather than a dense cockpit. Still, inputs and results are shown close together, with the result emphasized.
- Layout: On desktop, the calculator card sits in the main content with the right rail preserved via `Layout.astro`. On mobile, the main result is duplicated in a **sticky bottom bar** (handled inside `CalculatorIsland`) for quick visibility after scrolling.
- Interactions:
  - **Autocalculate** on input change (kg or unit switch).
  - A **Calculate** button is still present for accessibility and to support keyboard-only users.
  - **Reset** button clears all fields.
  - **Share** button copies the URL with query parameters (e.g., `?kg=70`).
- Progressive disclosure: Only a basic field is needed (kilograms). No advanced options are necessary, so no Basic/Advanced toggle is used.
- Mobile: `inputmode="decimal"` and `type="text"` are enforced via props to the island.
- Trust & clarity: Clear H1, short value prop, and a note that it uses the standard conversion factor (1 kg = 2.20462 lb). Includes theory, formula in LaTeX, real-world example, and 3 targeted FAQs.
- Structured data: `WebApplication`, `BreadcrumbList`, and `FAQPage` JSON-LD are injected for SEO.
