1) Thinking Process (UX & IA)

- UX archetype: This is a simple, single-purpose unit converter, so I’m using a **Wizard-lite / simple form** rather than a dense cockpit. Inputs and results are shown **side-by-side on desktop** and **stacked on mobile**.
- Progressive disclosure: Only the essential fields (centimeters and inches) are visible. A small “Advanced options” section exposes precision/rounding controls.
- Interaction:
  - **Auto-calculate** on input change.
  - A **Calculate** button is still present for accessibility and to support keyboard-only users.
  - **Reset** clears all fields and precision.
  - **Share** copies the URL with current values as query parameters.
- Mobile:
  - All numeric inputs use `inputmode="decimal"`.
  - A **sticky result bar** is assumed to be implemented inside `CalculatorIsland` for mobile when content is long; this page passes clear props to support that behavior.
- SEO & IA:
  - URL: `/en/science-and-mathematics/unit-conversion/cm-to-inches`
  - Rich content: theory, formula with LaTeX-style text, real-world example, and 3 focused FAQs.
  - JSON-LD: `WebApplication`, `FAQPage`, and `BreadcrumbList`.
- Visuals & microinteractions:
  - Clear H1 and value prop.
  - “Verified by” style trust note in text.
  - Tips/tooltips via helper text under inputs.
  - Result area visually highlighted; copy-ready output.
