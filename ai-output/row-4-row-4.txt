1) Thinking Process (UX & IA)

- UX archetype: Cockpit layout (not a multi-step wizard) because the conversion is instant, lightweight, and benefits from side-by-side inputs/results plus advanced options.
- Layout:
  - Desktop: Inputs on the left, live results on the right in a “result card”.
  - Mobile: Stacked layout with a sticky bottom result bar once there is a valid value.
- Progressive disclosure:
  - Basic: single main input (length in centimeters) with primary result in inches.
  - Advanced (toggle): show additional outputs (feet + inches breakdown, inches with different rounding, and quick reverse input for inches → cm).
- Interactions:
  - Autocalculate on every input change.
  - “Calculate” button kept as an explicit CTA (also useful for keyboard users).
  - “Reset” button clears all fields and returns to defaults.
  - “Share” button copies URL with current query params (?cm=VALUE&precision=VALUE).
  - Microinteractions: highlight result card on change, subtle focus styles on inputs.
- Mobile:
  - All numeric fields use inputmode="decimal".
  - Sticky result bar at bottom when form taller than viewport.
- SEO & IA:
  - URL: /en/science-and-mathematics/unit-conversion/cm-to-inches
  - Rich content: theory with formula in LaTeX, real-world example, 3 targeted FAQs.
  - JSON-LD: WebApplication + FAQPage + BreadcrumbList.
  - Application category: “ScienceApplication” (unit conversion in science/math context).
- Related tools:
  - Same cluster: other unit conversion tools (referenced generically in config snippet as required, without inventing specific slugs beyond what’s structurally needed).
  - Cross-cluster: not explicitly detailed in the row, so kept minimal and generic in config.
