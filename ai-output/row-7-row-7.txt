1) Thinking Process (UX & IA)

- UX archetype: Cockpit layout. The conversion is simple, but we want instant bidirectional conversion, unit switching, and educational content visible at once. Cockpit works better than a step-by-step wizard here.
- Layout: On desktop, inputs and results are side-by-side in a compact panel; on mobile, stacked with a sticky result bar at the bottom once the form exceeds viewport height.
- Inputs:
  - Two primary fields: angstroms → nanometers and nanometers → angstroms, both with `inputmode="decimal"`.
  - Progressive disclosure: an “Advanced options” toggle for secondary helpers (e.g., scientific notation display).
  - Autocalculate on input change; a “Calculate” button is still present for accessibility and explicit control.
- Microinteractions:
  - Subtle highlight when results change.
  - Focus styles on inputs.
  - Contextual validation messages instead of generic “Error” (e.g., “Value must be a finite number”).
- Utility:
  - Reset button clears all fields and state.
  - Share button copies the current URL with query parameters representing the current inputs.
- SEO & IA:
  - URL: `/en/science-and-mathematics/unit-conversion/angstrom-to-nm`.
  - Rich content: theory with LaTeX, real-world example, and 3 targeted FAQs.
  - JSON-LD: `WebApplication`, `FAQPage`, and `BreadcrumbList`.
  - Application category: `ScienceApplication` (scientific unit conversion).
- Right rail: Uses `Layout.astro` with sidebar preserved for ads.
- Related tools: Within the same “Unit Conversion” cluster; we only reference the BTU→kWh sample as a related unit converter in config (as a stand-in pattern), but do not invent new tools beyond what’s implied by the project (kept minimal and generic).
