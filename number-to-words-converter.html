The Formula Explained

Let N ∈ ℤ, with sign s ∈ {−1, 0, +1}. Let ⌊·⌋ be floor.
Let parts be the base-1000 groups of |N|: g₀ (units), g₁ (thousands), g₂ (millions), …

For each group g = 0…:
  g = 100·a + 10·b + c, a,b,c ∈ {0…9}

Number words per group:
- Hundreds: a > 0 ⇒ “{ones[a]} hundred”
- Tens/ones:
  if b = 1 ⇒ “{teens[c]}”
  else     ⇒ “{tens[b]}{(c>0 ? (hyphen? '-' : ' ') + ones[c] : '')}”

British “and” rule:
- If (a>0) and (b>0 or c>0) and britishAnd ⇒ insert “and” between hundreds and tens/ones.

Scale suffix:
- Short scale: names[k] = thousand (10³), million (10⁶), billion (10⁹), …
- Long scale:  names[k] = thousand (10³), million (10⁶), milliard (10⁹), billion (10¹²), …

Decimal:
- Non-currency: speak as digits after “point” (e.g., 12.305 → “twelve point three zero five”).
- Currency: use fractional “and xx/100” for two decimal places.

Final:
- Join non-zero groups from highest to lowest with separators (commas).
- If s = −1 ⇒ prefix “minus”.

This algorithm follows standard English naming with selectable scale and style toggles.

Glossary of Variables

  • Scale: Naming system for large numbers (short vs long).
  • British “and”: Inserts “and” between hundreds and tens/ones (e.g., “one hundred and five”).
  • Hyphenate: Prints a hyphen for 21–99 (per GPO style), e.g., “twenty-one”.
  • Currency: Formats output as cheque-style text for USD/EUR/GBP.
  • Words: Final spelled-out representation of the input number with selected options.

How It Works: A Step-by-Step Example

  1. Input: 12,345.67
  2. Options: Scale = Short; British “and” = On; Hyphenate = On; Currency = USD.
  3. Groupings: 12 | 345 → “twelve thousand” + “three hundred and forty-five”.
  4. Currency rule: “dollars and 67/100”.
  5. Result: Twelve thousand three hundred and forty-five dollars and 67/100.

Frequently Asked Questions

What’s the difference between short and long scale?

Short scale uses billion = 109 and trillion = 1012. Long scale inserts milliard at 109 and uses billion = 1012.

Does the tool follow hyphenation rules?

Yes. When enabled, it hyphenates twenty-one to ninety-nine per GPO Style Manual §6.36–6.37.

How are decimals handled?

Non-currency decimals are read digit by digit after “point”. In currency mode, they are rendered as “and xx/100”.

Is “and” before the tens optional?

Yes. Toggle British “and” on to produce “one hundred and one”.

What range is supported?

Up to 10303 (centillion) on short scale; corresponding long-scale names are available.

Can I convert negative numbers?

Yes. The output is prefixed with “minus”.
