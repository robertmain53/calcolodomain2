How the IP calculator works

This IP calculator focuses on IPv4 and follows the same logic network engineers use when doing subnetting by hand. It converts the dotted-decimal IP address and subnet mask into 32-bit integers, applies bitwise operations, and then converts the results back to human-readable form.

Key formulas

1. Convert dotted decimal to 32-bit integer

IP = a·256³ + b·256² + c·256 + d

where a.b.c.d is the IPv4 address.

2. Network and broadcast address

network = ip & mask

broadcast = network | (~mask)

3. Number of addresses and usable hosts

total = 2^(32 − prefix)

usable = total − 2    (for prefixes ≤ 30)

For /31 and /32, all addresses are treated as usable (special point-to-point and host-only cases).

4. Wildcard mask

wildcard = 255.255.255.255 − subnet mask

Address class and private ranges

Historically, IPv4 addresses were grouped into classes:

  • Class A: 0.0.0.0 – 127.255.255.255 (default /8)
  • Class B: 128.0.0.0 – 191.255.255.255 (default /16)
  • Class C: 192.0.0.0 – 223.255.255.255 (default /24)
  • Class D: 224.0.0.0 – 239.255.255.255 (multicast)
  • Class E: 240.0.0.0 – 255.255.255.255 (experimental)

Today we mostly use CIDR, but class ranges are still useful for quick mental checks.

The calculator also detects whether the address is in a private range:

  • 10.0.0.0 – 10.255.255.255 (10.0.0.0/8)
  • 172.16.0.0 – 172.31.255.255 (172.16.0.0/12)
  • 192.168.0.0 – 192.168.255.255 (192.168.0.0/16)

Reverse DNS (in-addr.arpa)

Reverse DNS for IPv4 uses the in-addr.arpa domain. The IP octets are reversed and .in-addr.arpa is appended. For example:

192.0.2.10 → 10.2.0.192.in-addr.arpa

This name can then be looked up with a PTR record to map an IP back to a hostname.

Worked examples

Example 1 – 192.168.1.10/24

  • IP: 192.168.1.10
  • Mask: 255.255.255.0 (/24)
  • Network: 192.168.1.0
  • Broadcast: 192.168.1.255
  • Usable hosts: 192.168.1.1 – 192.168.1.254 (254 hosts)

Example 2 – 10.0.0.1/30

  • Mask: 255.255.255.252 (/30)
  • Network: 10.0.0.0
  • Broadcast: 10.0.0.3
  • Usable hosts: 10.0.0.1 – 10.0.0.2 (2 hosts)

FAQ

Can I use this IP calculator for IPv6?

This tool is optimized for IPv4. IPv6 subnetting uses different rules and much larger address space. For IPv6 you typically work with /64 and /48 prefixes and do not have broadcast addresses.

Why do /31 and /32 behave differently?

In traditional IPv4 subnets, the first and last addresses are reserved as network and broadcast. However, RFC 3021 allows /31 subnets for point-to-point links, effectively giving two usable addresses. A /32 represents a single host address. The calculator handles these cases by treating all addresses as usable.

What is the wildcard mask used for?

Wildcard masks are mainly used in router and firewall ACLs (for example, Cisco IOS). They specify which bits of an address must match and which bits can vary. A 0 bit means “must match”, and a 1 bit means “don’t care”.
