How this Prim’s algorithm calculator works

Prim’s algorithm finds a minimum spanning tree (MST) of a connected, weighted, undirected graph. The MST connects all vertices with the minimum possible total edge weight and no cycles.

Algorithm steps (high level)

  1. Pick a start vertex and mark it as visited.
  2. Among all edges that connect a visited vertex to an unvisited vertex, pick the one with the smallest weight.
  3. Add that edge and the new vertex to the tree.
  4. Repeat step 2–3 until all vertices are included (or no more edges are available).

This tool implements the classic O(V²) version of Prim’s algorithm using an adjacency matrix internally. For educational graph sizes (up to 15 vertices here), this is fast and easy to follow.

Input format details

  • Vertices: numbered from 1 to n.
  • Edges: each line is u v w where:
    • u and v are vertex indices (integers).
    • w is the edge weight (positive number).
    • The graph is treated as undirected, so u v w is the same as v u w.

What you see in the visualization

  • Nodes: vertices laid out in a circle for readability.
  • Gray edges: all edges in the original graph.
  • Blue edges: edges that are part of the MST.
  • Highlighted edge: the edge chosen in the current step.
  • MST table: lists each selected edge and its weight.
  • Step log: textual explanation of each iteration (which edge was considered, chosen, or skipped).

When Prim’s algorithm may fail or be incomplete

  • If the graph is disconnected, Prim’s algorithm can only build an MST for the connected component containing the start vertex. This tool will warn you if some vertices remain unreachable.
  • If there are no edges or only one vertex, the MST is empty and the total weight is 0.

Prim’s algorithm vs. Kruskal’s algorithm

  • Prim’s: grows a single tree from a start vertex, always adding the cheapest edge from the tree to a new vertex.
  • Kruskal’s: starts with all vertices separate and repeatedly adds the globally cheapest edge that doesn’t form a cycle, effectively merging components.

Both produce a minimum spanning tree, but they differ in implementation details and performance depending on graph density and data structures.

Frequently asked questions

Can I use negative edge weights?

Yes, Prim’s algorithm still works with negative edge weights as long as the graph is connected and undirected. This calculator accepts negative weights, but be aware that some textbook examples assume non‑negative weights.

What is the output of this calculator?

The tool returns:

  • The list of MST edges (u, v, w).
  • The total MST weight.
  • A step‑by‑step log of how Prim’s algorithm progressed.
  • A visual graph with MST edges highlighted.

How large a graph can I enter?

For clarity and performance in the browser, this calculator supports up to 15 vertices. That’s more than enough for most homework, exam practice, and interview examples. For very large graphs, you’d typically run Prim’s algorithm in a programming language instead of a visual tool.
