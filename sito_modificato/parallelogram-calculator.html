The formulas explained

Area (base & height): \\( A = b \\, h \\)
Area (sides & angle): \\( A = a \\, b \\, \sin\\theta \\)
Perimeter: \\( P = 2(a + b) \\)
Heights: \\( h_b = a\\sin\\theta, \\quad h_a = b\\sin\\theta \\)
Diagonals: \\( p = \sqrt{a^2 + b^2 + 2ab\\cos\\theta}, \\quad q = \sqrt{a^2 + b^2 - 2ab\\cos\\theta} \\)
Angles: \\( \\alpha = \\theta, \\quad \\beta = 180^{\\circ} - \\theta \\) (degrees)
Coordinates (vectors \\(\\vec{AB}\\), \\(\\vec{AD}\\)): \\( A = |\\vec{AB} \\times \\vec{AD}| = |x_{AB}y_{AD} - y_{AB}x_{AD}| \\)

How it works: a step-by-step example

Sides + Angle example

Given \\(a = 7\\), \\(b = 10\\), \\(\\theta = 40^{\\circ}\\):

  • \\(A = a b \\sin\\theta = 7 \\cdot 10 \\cdot \\sin 40^{\\circ} \\approx 45.0\\)
  • \\(P = 2(a+b) = 34\\)
  • \\(h_b = a\\sin\\theta \\approx 4.50\\); \\(h_a = b\\sin\\theta \\approx 6.43\\)
  • \\(p = \\sqrt{a^2+b^2+2ab\\cos\\theta} \\approx 15.23\\); \\(q \\approx 6.73\\)
  • \\(\\alpha = 40^{\\circ}, \\; \\beta = 140^{\\circ}\\)

Frequently asked questions

Is Base & Height enough to get everything?

It determines the area, but not the entire shape. Add side a (or the angle) to compute perimeter, angles, and diagonals.

What angle range is valid?

The included angle must be between 0° and 180°, exclusive. Parallelograms cannot have 0° or 180° internal angles.

Do diagonals bisect each other?

Yes. In any parallelogram, the diagonals bisect each other, though their lengths are generally different unless it’s a rectangle.

How does the coordinate mode compute area?

It uses vectors \\(\\vec{AB}\\) and \\(\\vec{AD}\\) and takes the magnitude of their 2D cross product.

What about rhombus/rectangle/square cases?

All included: set \\(a=b\\) for a rhombus; set \\(\\theta=90^{\\circ}\\) for a rectangle; both for a square.
