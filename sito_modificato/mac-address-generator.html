How the MAC address generator works

A MAC address is a 48-bit value usually written as six octets in hexadecimal: AA:BB:CC:DD:EE:FF. This tool generates valid MAC addresses and lets you control the important bits in the first octet:

  • Unicast vs multicast – least significant bit of the first octet.
  • Universal vs locally administered – second least significant bit of the first octet.
  • Vendor prefix (OUI) – the first 3 bytes, if you provide one.

Bit layout of the first octet

First octet bits: b7 b6 b5 b4 b3 b2 b1 b0

b1 (U/L bit): 0 = universally administered, 1 = locally administered
b0 (I/G bit): 0 = unicast (individual), 1 = multicast (group)

When you choose Unicast + Universal, the generator ensures both bits are 0. For Locally administered multicast, both bits are set to 1, and so on.

Using an OUI (vendor prefix)

If you enter a 3-byte OUI such as 00:1A:2B or 001A2B, the generator keeps those 3 bytes and randomizes the remaining 3 bytes. It will still adjust the first octet’s two least significant bits to match your unicast/multicast and universal/local choices.

Typical use cases

  • Assigning MACs to virtual machines, containers, or KVM/libvirt guests.
  • Generating test data for network simulations and labs.
  • Creating randomized MACs for privacy or QA environments.

MAC address formats supported

  • Colon separated: AA:BB:CC:DD:EE:FF
  • Dash separated: AA-BB-CC-DD-EE-FF
  • Cisco dotted: AABB.CCDD.EEFF
  • Plain hex: AABBCCDDEEFF
  • Lowercase colon: aa:bb:cc:dd:ee:ff

FAQ

Are the generated MAC addresses unique?

The generator uses cryptographically strong randomness where available (window.crypto.getRandomValues) and the 48-bit space is huge (over 281 trillion possibilities). Collisions are extremely unlikely, but uniqueness cannot be mathematically guaranteed. For safety, avoid reusing generated MACs across different networks if you export them to production.

Can I generate multicast MAC addresses?

Yes. Select Multicast under address type. The generator sets the I/G bit (b0) of the first octet to 1. Note that multicast MACs are typically derived from higher-layer multicast addresses (e.g., IPv4 224.0.0.0/4 or IPv6 ff00::/8), so use them carefully.

What is a locally administered MAC address?

Locally administered addresses have the U/L bit (b1) set to 1. They are not tied to a vendor’s OUI block and are meant to be assigned by administrators or software. Operating systems often use locally administered MACs for randomized Wi‑Fi addresses and virtual interfaces.

Is it safe to use this tool for production networks?

For physical NICs, you should normally rely on vendor-assigned MAC addresses. This tool is ideal for labs, testbeds, and virtual environments. If you do use generated MACs in production, ensure they are unique within each Layer 2 broadcast domain.
